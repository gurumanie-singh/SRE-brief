Interested in a modern way to manage Red Hat Enterprise Linux (RHEL)? Image mode is a deployment option that allows the operating system to be built, deployed, and updated like a container image. A key reason to deploy from a bootc image is its enhanced consistency and reliable, atomic updates and rollbacks.

Technical breakdown

Image mode is a deployment option that allows the operating system to be built, deployed, and updated like a container image.

With RHEL 10, you can install that image from the network using the same installer you already use for traditional RHEL deployments: Anaconda.

Impact

What the bootc command does From a user perspective, the bootc command achieves the same result as the older ostreecontainer command: Installing an immutable system from a container image. Once this step is complete, Anaconda continues with the final configuration steps — including user accounts, root password, SSH keys, and other kickstart options you specified — ensuring the installed system is fully set up before the first boot. You can combine bootc with typical kickstart commands, such as autopart , clearpart , reqpart , part , network , rootpw , user , timezone , %pre or %post . User account requirements: Either set a root password with the rootpw kickstart command or define an administrator user, so that the installation can proceed.

Classified impact band: medium. Affected layer signal: networking.

Resolution / fix

A key reason to deploy from a bootc image is its enhanced consistency and reliable, atomic updates and rollbacks.

The source image reference must be specified with a registry: prefix, whereas that prefix must NOT be present in the target image reference.

What to keep in mind A few limitations are worth noting before deployment:

Image reference format: Both the --source-imgref and --target-imgref options are currently required, and their usage requires a specific format: the registry: prefix must be present for the source image reference but excluded from the target image reference.
