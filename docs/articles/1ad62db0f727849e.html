Overview

The web has always had to adapt to new standards. It learned to speak to web browsers, and then it learned to speak to search engines. Now, it needs to speak to AI agents.

Technical breakdown

Integrating agent readiness into Cloudflare URL Scanner Cloudflare's URL Scanner lets you submit any URL and get a detailed report on it: HTTP headers, TLS certificates, DNS records, technologies used, performance data, and security signals.

Think of it as a sitemap written for an LLM to read rather than a crawler to index:

# My Site > A developer platform for building on the edge. ## Documentation - [Getting Started](https://example.com/docs/start.md) - [API Reference](https://example.com/docs/api.md) ## Changelog - [Release Notes](https://example.com/changelog.md) Markdown content negotiation goes even further.

Unlike links buried inside HTML, the Link header is part of the HTTP response itself, which means an agent can find links to resources without having to parse any markup:

Lessons learned

It learned to speak to web browsers, and then it learned to speak to search engines.

There’s a better way that allows humans to explicitly grant access: sites that support OAuth can tell agents where to find the authorization server ( RFC 9728 ), allowing agents to send humans through an OAuth flow, where they can choose to properly grant access to the agent.

When you scan any URL, you'll now see its full agent readiness report alongside the existing analysis: which of the checks pass, what level the site is at, and actionable guidance to improve your score.

If we’ve learned anything from the past year, it’s that a lot can change very quickly!
