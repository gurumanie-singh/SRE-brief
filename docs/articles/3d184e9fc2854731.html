Technical breakdown

These are not terminal commands; they are structured tool calls that Anthropic's harness sends to your worker through the work queue:

[tool_use] bash: python3 dispatch.py metrics_agent "Check checkout-service latency, error rates, resource usage" [tool_use] bash: python3 dispatch.py log_agent "Search checkout-service logs for errors since 14:30 UTC" [tool_use] bash: python3 dispatch.py runbook_agent "Find past incidents matching connection pool exhaustion" The worker pod receives these tool calls from the queue.

Root cause highlight

Likely root-cause theme: cascading failure. A single sandbox cannot restrict access to only one of those systems because the same process needs all three. Now each sandbox can enforce least privilege because each agent only handles one specific task. The runbook agent, which only needs read-only file access, can reach the Prometheus API because the metrics agent needs it.
