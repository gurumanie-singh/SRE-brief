Overview

AI applications often call model provider APIs such as OpenAI, Anthropic, or Google directly from the application code. In many cases, the application contains the logic for calling the provider endpoint, handling authentication, and formatting requests. This works well when an application relies on a single provider.

Root cause highlight

Likely root-cause theme: control plane issue. Because the application interacts only with the gateway, you can switch providers without changing the application code. A team might choose one provider for reasoning tasks, another for multimodal use cases, and self-hosted models for workloads that require more control over cost, performance, or data locality. This introduces a centralized control plane for model access while applications continue using a single OpenAI-compatible API endpoint.
