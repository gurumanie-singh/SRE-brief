Technical breakdown

We identified the three async patterns responsible for the majority of test flakiness (a double-retry race condition, debug logging that polluted test output and arbitrary wait timers that masked real timing bugs) and banned all three at the lint level.

A feature flag controls the switch.

Rollback is a flag flip.

Edge cases added under deadline pressure.

We found several of these: a default group behavior tied to a specific API interaction, permission edge cases that had grown organically, features added in earlier development cycles that hadn't yet been covered by tests.

Impact

We added three more rules:

Require a shared utility for every table Prevent direct user identity lookups that bypass the shared utility Restrict API client imports to the data layer only. Nine months later, we'd replaced the entire stack in a zero-downtime migration. Permission checks were baked into the platform shell (a system we call "Chrome"), meaning the UI asked the shell, "Can this user do X?" and trusted whatever came back. Customers on V1 see no change.

Root cause highlight

Likely root-cause theme: deployment failure. Some regressions still shipped, because no test suite catches everything. The distinction matters because accountability should be specific. I built the infrastructure because the alternative was shipping something I couldn't stand behind.
